  • a:hoverの盲点
    cssを書いていてa:hoverの内容が適用されなくなった。
    a:hoverで文字色を変更しようとしたところ、プロパティが効かなくなった。
    リセットCSSの悪さかと考え、リセットCSSをの@import行をコメントアウトしたが変化なし。
    原因
    cssの記述におけるルールとして、プロパティを適用する際に優先順位を決める為に「点数」が設けられている。
    この点数が同点の場合の優先順位は後述された方が優先されるということ。
    それを踏まえて、なぜa:hoverが機能しなかったのか。
    a:visitedがa:hoverよりも後述されていた為にa:visitedのプロパティが優先して適用されてしまっていた。
                                        .refelence_url li a:hover{
                                        color:#fff;
                                    }
                                    .refelence_url li a:visited{
                                        color:#aaa;
                                    }
    確認したところ、実際に上述と同じ書き方をしていた。
    a:visitedを入れ替えてみたところ、期待した挙動になった。

  • background-imageのみの透過について
    そういえばbackground-imageのみの透過ってできるんか？
    指定した要素に対して不透明度を設定するプロパティがopacty:hoge;なんだけど、こいつでbackground-imageを不透明度の調整をしようとした。
    結果としては不可能だった。そんな気はしていたけど。
    そもそも「指定した要素に対して不透明度を設定するプロパティ」であって、プロパティを調整するプロパティではないもんね。仕方ないね。
    でも、なんかできそうだから少しだけ調べたら、それっぽくできる方法あったわ。
    やり方のイメージとしてはPhotoshopにおいてbackground-imageレイヤーの上層に白ベタのレイヤーを1枚乗せる。
    そして、その乗せた白ベタレイヤーの不透明度を調整して、擬似的にbackground-imageが透過したように見せる。
                                    hoge{
                                    background-image:url("hogehogehoge.png");
                                    background-color:rgba(255,255,255,0.6);
                                    background-blend-mode:hard-light;
                                }
    こんな感じ。
    background-blend-modeのプロパティ値はなんか色々あるみたいだから、調べてみなさい。てかPhotoshopの描画モードとほぼ同義っぽいぞ。
